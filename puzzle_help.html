Basic Usage

The Twisty Puzzle Page presents its user with any one of several twisty puzzles which can be selected from a scrolling menu revealed when the mouse cursor nears the top of the page. Most of these are well-known puzzles; others may have been made up for fun. Whatever the case, all puzzles are manipulated the same way using the mouse. Left-click and drag to orient the puzzle. Hover over the puzzle until a desired sub-set of its faces are highlighted, then use the mouse wheel to rotate those faces. If they don't rotate, then it's due to bandaging of the puzzle.

Double-clicking anywhere on screen will scramble the puzzle with your confirmation.

For puzzles that have a special kind of move, such as the Curvy-Copter, hold the Shift or Ctrl key down while rotating the mouse wheel to take advantage of these types of moves.

Note that some axes of a puzzle are used to rotate more than one layer of a puzzle. The hot-spot for the default layer should be clear. A secondary or tertiary layer, however, can be rotated by hovering the mouse further away from the puzzle, but along the same axis. This takes a bit of patience to manage, but isn't too annoying in practice. See the 4x4 Rubik's Cube for an example of what I'm talking about here.

In the lower left-hand corner, you'll find a few read-outs and controls. All puzzles can be textured. This makes the puzzle a harder challenge in most cases. A mirror-reflection of the puzzle can be toggled. I'm not sure how helpful this really is, but it's there. Full undo/redo is supported. You can save and restore the state of any puzzle under a given name.

Known Issues

Here is a list of currently known bugs/issues with the software...

  • Undo/redo state is not yet saved along with the state of the puzzle.
  • Some puzzles currently have incorrect border geometry.
  • The sequence feature has some bugs and is currently only useful for puzzles of cube shape.
  • Highlighted faces can distract you from the color scheme of the cube. Perhaps only border colors should be used to highlight faces.
  • Some puzzles requiring many colors do not have a diverse enough color scheme. Consequently, some colors are too close together to be discerned easily.
  • The mouse-driven interface can be a bit hard to use in some cases, such as when a back-facing part of the puzzle is accidentally rotated. Perhaps only forward-facing parts should be selectable.
  • Hot-spots for face selection are not all optimally tuned. Consequently, it can be hard to know where to hover the mouse to highlight faces.
  • Not all my favorite puzzles are supported, nor might the puzzle engine be robust-enough or general-purpose-enough to handle them all.
  • Not all puzzles can be sufficiently scrambled using the naive algorithm I have in place for that purpose.
  • The approximate cutting of rounded puzzle faces means that after a rotation, edges of these faces may not line up so nicely next to others.
  • Last but not least, no one gives a rat's butt about my puzzle page, but that's okay. I wrote it for me, and it's helped me learn some web development.
